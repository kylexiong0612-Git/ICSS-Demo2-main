# 前端开发规范-移动端

## 1. 项目概述

### 1.1 技术栈

* **核心框架**: Vue 3.5.13 (使用 `<script setup>` 语法糖)

* **构建工具**: Vite 5.1.4

* **UI组件库**: Vant 4.8.5 (专为移动端设计)

* **状态管理**: Pinia + pinia-plugin-persistedstate

* **路由管理**: vue-router 4.3.0

* **网络请求**: axios 1.8.4

* **样式处理**: Sass + PostCSS (使用 px 转 rem 实现移动端适配)

### 1.2 项目结构

```
src/
├── api/                 # API接口定义
├── assets/              # 静态资源文件
├── components/          # 可复用组件库
├── router/              # 路由配置
├── stores/              # 状态管理
├── utils/               # 工具函数库
├── views/               # 业务视图组件
├── App.vue             # 根组件
└── main.js             # 入口文件
```

### 1.3 核心目录概述

#### 1.3.1 /src/router - 路由配置

路由使用 Vue Router 实现，主要配置在 \[src/router/index.js]中。路由配置包含页面路径、组件映射以及权限控制

#### 1.3.2 /src/stores - 状态管理

使用Pinia进行状态管理

#### 1.3.3 /src/api - 接口请求

所有接口请求统一定义在 \[src/api/]目录下，通过 \[src/util/http.js]封装 axios 实现统一的请求拦截和错误处理。

### 1.4 环境配置

项目支持多种环境配置：

* `.env.development` - 开发环境

* `.env.sit` - sit测试环境

* `.env.uat` - uat测试环境

* `.env.production` - 生产环境

不同环境下可通过环境变量配置API地址等参数。

### 1.5 移动端调试工具

vconsole是一个轻量级移动端调试工具，可以查看页面布局、网络请求、错误信息等等。\
在main.js中按需引入vconsole

```javascript
// 只有测试环境开启vconsole
if (import.meta.env.VITE_ENABLE_CONSOLE === 'true') {
  window.vConsole = new vconsole()
}
```

## 2 UI规范

### 2.1 移动端适配方案

#### 2.1.1 尺寸适配

该项目采用了基于 amfe-flexible 和 postcss-pxtorem 的移动端适配方案：

* 使用 amfe-flexible 动态设置根元素字体大小，实现移动端适配

* 通过 postcss-pxtorem 插件将 px 单位自动转换为 rem 单位

* 根字体大小设置为 37.5px，适配设计稿宽度为 375px 的移动端设计

这样开发者可以直接使用设计稿上的 px 尺寸，构建工具会自动完成单位转换，实现良好的跨设备适配效果。

#### 2.1.2 全面屏适配

项目支持全面屏适配，通过设置 viewport-fit=cover 来适配全面屏。

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0, user-scalable=no, viewport-fit=cover">
```

同时需配合 CSS 安全区域（safe area）处理,避开顶部刘海区、底部Home条区域。

```css
.safe-area-inset-top { 
   padding-top: constant(safe-area-inset-top);
   padding-top: env(safe-area-inset-top);
}
.safe-area-inset-bottom { 
   padding-bottom: constant(safe-area-inset-bottom);
   padding-bottom: env(safe-area-inset-bottom);
}
```

### 2.3 布局规范

页面主要分为列表页、详情页两种布局，所有页面顶部固定为标题栏

##### 2.3.1 标题栏

页面标题栏固定在页面顶部，需考虑iOS上顶部安全距离适配，高度默认48px，左右内边距16px，内容垂直方向上居中，标题文字水平居中，返回按钮靠左显示颜色为灰色，右侧按钮可选，文字颜色为主题色。使用<van-nav-bar>组件。

##### 2.3.2 底部操作栏

页面如包含底部操作栏，则底部操作栏固定在页面底部，需考虑iOS上底部安全距离适配，高度默认60px，左右16px内边距。其中基础按钮使用<van-button>组件，normal尺寸，无圆角，水平排列，垂直方向和水平方向上都居中，如有多个按钮，按钮左右之间保持16px的距离，根据按钮含义搭配使用朴素按钮与默认按钮。

##### 2.3.3 详情页

* 1、页面主体内容占据除去标题栏和底部操作栏后的页面剩余高度，高度超出时垂直方向可滚动，宽度100%，上下外边距16px，字体尺寸14px，有背景色, 无特殊说明则所有元素无圆角。

* 2、页面主体内容垂直方向上分为不同的区域，每块区域有自己的标题栏，标题栏默认高度40px, 宽度100%, 左右16px内边距，内容垂直居中。标题内容水平方向依次排列：图标、标题文字、备注(可选)、标题按钮（可选）。

  * 图标颜色为主题色；

  * 标题文字加粗、黑色字体、宽度width:fit-content；

  * 备注为灰色字体紧跟标题；

  * 按钮颜色为主题色并靠右显示。

* 3、每个区域内如果有多个输入框则使用一个<van-cell-group>包裹多个<van-field>，显示内边框。

  * 表单项如无特殊说明，不要添加required属性，不要显示必填星号；

  * 表单的输入框文本位置默认左对齐，设置合适的label-width宽度使得输入框文本不换行显示

  * 表单通过设置右对齐input-align="right"实现内容右对齐；

  * 如果是复选框类型的表单项，则每行排列2个表单项，也应该实现右侧对齐。

  ```html
  <!-- 表单布局示例代码 -->
  <van-form label-width="100px" input-align="right">
     <van-cell-group>
        <van-field
           v-model="form.policyNo"
           name="policyNo"
           label="保单/投保单号"
        />
        <van-field
           v-model="form.name"
           name="name"
           label="姓名"
        />
     </van-cell-group>
  </van-form>
  ```

##### 2.3.4 列表页

* 1、页面如有搜索栏，则排列在标题栏下方，和标题栏一起固定在页面顶部。宽度100%，使用<van-search>组件，使用action插槽自定义右侧【搜索】文字按钮,示例代码如下。

```html
<van-search
  v-model="value"
  show-action
  placeholder=""
  @search="onSearch"
>
  <template #action>
    <div @click="onSearch">搜索</div>
  </template>
</van-search>
```

* 2、页面如有Tab标签页，则排列在搜索栏下方，和标题栏、搜索栏一起固定在页面顶部。使用<van-tabs>组件。标签页内容另外自行实现。

* 3、页面flex垂直布局，需固定在页面顶部的元素使用一个父元素包裹。

* 4、页面主体内容占据除去标题栏、搜索栏、标签页栏的页面剩余高度，flex:1 ,高度超出时垂直方向可滚动，宽度100%，上下外边距16px，字体尺寸14px，有背景色, 无特殊说明则所有元素无圆角。

* 5、页面主体内容使用列表组件包裹，滚动分页，使用<van-list>组件实现。

* 6、每个列表项是无圆角卡片风格，宽度100%高度自适应

  * 卡片之间上下间距12px，白色背景；

  * 卡片内边距16px，卡片内的信息项上下依次展示；

  * 每个信息项字体尺寸14px，行高1.5，上下内边距7px；

  * 信息项描述左侧对齐，字体颜色为#666666，最小宽度90；信息项内容右侧对齐，字体颜色为#333333。

  * 如果信息项可点击，则信息项内容右侧展示右箭头图标，图标颜色为#999999。

  * 不要使用<van-cell-group>组件实现。

### 3 编码规范

#### 3.1 Vue 组件开发

1. 使用 Vue 3 的 Composition API (script setup) 语法
2. 模板(template)、脚本(script)、样式(style)三部分清晰分离
3. 按照功能或模块组织文件，组件引用按需导入
4. 样式使用 scoped 防止样式污染

#### 3.2 CSS/SCSS 规范

1. 使用 SCSS 编写样式，确保移动端适配
2. 实现响应式设计，适配不同尺寸的移动设备
3. 代码缩进统一为 2 个空格
4. 页面中的按钮、跳转链接、图标如果没有特殊说明，一律采用主题色：#00a758;

#### 3.3 JavaScript 规范

1. 代码缩进统一为 2 个空格
2. 组件属性按字母顺序排列
3. 方法按功能相关性分组

#### 3.4 状态管理规范

1. 表单数据使用 Vue 的 ref 进行管理
2. 计算属性使用 computed 实现
3. 所有用户输入需要进行数据绑定
